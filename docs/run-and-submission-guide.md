This guide explains how to run the project end-to-end, what the original assignment asked for, and where every submission deliverable lives. All submission-critical deliverables (spec 003) are now implemented and verified — this guide documents the finished pipeline, not a plan to build it.

## 1. Assignment Summary

Source brief: `docs/assignment/Cinnamon_Export_Sales_Forecasting_Assignment (1).docx`

The assignment gives about 60,000 transaction-level rows from a Sri Lankan cinnamon export company, spanning early 2022 to September 2025. Each row is one sales transaction. The target variable is `Sales Qty`.

Core objectives:

- Forecast weekly sales quantity per product for the next 12 weeks.
- Identify products by the first 9 characters of `Product Code`.
- Aggregate raw transactions into weekly time series before modeling.
- Produce country-level forecasts for selected product-country drill-downs.
- Handle both high-volume products and sparse/intermittent low-volume products.

Evaluation areas:

- Forecasting performance using error metrics such as RMSE, MAE, MAPE, or better sparse-safe alternatives.
- Training efficiency: how long models take to train and what resources they use.
- Forecasting speed: how long future prediction generation takes.
- Model transparency and insights:
  - training/validation loss or accuracy curves;
  - correlation matrix for engineered features such as day of week, week of month, seasonality, and holidays;
  - feature importance;
  - sparse/zero-sales week handling;
  - year-over-year and seasonal trend visualizations.

Required deliverables:

- Source code for preprocessing, feature engineering, model training, and forecasting.
- Forecast output file, CSV or Excel, containing 12-week forecasts for each product.
- Project report covering data exploration, preprocessing, model selection, training method, metrics, comparisons, visualizations, and challenges.
- Brief presentation summarizing approach, findings, and results.

Important assignment notes:

- Raw data is transaction-level and must be aggregated.
- Low-volume products need different handling from high-volume products.
- Data quality must be checked: missing values, outliers, date inconsistencies.
- Feature engineering is expected: lag features, rolling averages, seasonal indicators, zero-streak length, and similar features.

## 5. Run Everything End-to-End

The single command that reproduces every artifact from raw data through final figures and notebooks:

```bash
make all
```

This runs, in order: `pipeline` (Stages 2–6) -> `test` (pytest suite) -> `forecast` (models, backtest, forecast assembly) -> `report` (figures) -> `notebooks` (execute in place). It takes about 2 minutes on a normal laptop.

To run stages individually:

```bash
make pipeline      # clean-data -> scope -> weekly -> features -> tiering
make test          # pytest suite
make forecast      # high_volume -> sparse -> country_demo -> backtest -> assemble_forecast
make report        # correlation/SHAP/loss/actual-vs-predicted/sparse-zero/YoY figures
make notebooks     # execute all notebooks in place
```

Or a single pipeline/model stage directly:

```bash
make clean-data
make scope
make weekly
make features
make tiering
make verify        # re-runs `make pipeline`'s built-in assertions
```

Remove generated processed data and outputs, then rebuild:

```bash
make clean
make all
```
